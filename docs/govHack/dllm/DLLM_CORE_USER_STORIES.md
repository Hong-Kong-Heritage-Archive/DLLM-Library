# 📚 DLLM 核心功能 - User Stories

## 🎯 **專案概述**

**系統名稱**: DLLM (Distributed Local Library Module)  
**核心概念**: 去中心化本地圖書共享社區  
**主要功能**: 書籍上傳/放置、借閱管理、社區圖書流通

## 🚀 **系統願景**

DLLM 是一個開源的去中心化圖書共享平台，旨在建立一個去中心化的本地圖書共享社區。它並非要取代傳統圖書館，而是要彌補其在空間、管理及藏書種類上的不足。

**核心概念**: 採用「chain 借」（連鎖借閱）概念，讓書籍透過「書主—持有人—借書人」的多點傳遞模式，在社區中更自由、靈活地流通。

**開發理念**: 強調透過真實數據測試來驅動開發，並積極與社區組織連結，設立實體交收點，將線上系統與線下人際網絡結合。

**開源精神**: 採用開源（Open Source）模式，讓任何社區都能建立自己的版本，避免單一平台壟斷，實現一個真正由下而上、「無大台」的知識共享網絡。

## 👥 **主要用戶角色 (User Personas)**

### **1. 書主 (Book Owners)**
- **年齡**: 25-70歲
- **技術水平**: 中等
- **動機**: 分享知識，延長書籍生命週期，讓珍貴或絕版書有更多機會在社區流通
- **痛點**: 擔心書籍遺失，缺乏借閱管理工具，需要保障私隱的靈活性

### **2. 借書人 (Book Borrowers/Requesters)**
- **年齡**: 18-65歲
- **技術水平**: 基礎到中等
- **動機**: 獲得學習資源，參與社區知識共享，解決傳統圖書館空間有限、程序繁複、無法觸及罕有書籍等問題
- **痛點**: 不知道社區內有什麼書籍可用

### **3. 書籍持有人 (Book Holders)**
- **年齡**: 20-60歲
- **技術水平**: 基礎到中等
- **動機**: 暫時保管書籍，協助社區流通，參與「書主—持有人—借書人」的多點傳遞模式
- **痛點**: 需要清楚了解保管責任和期限

### **4. 社區組織者 (Community Organizers)**
- **年齡**: 30-60歲
- **技術水平**: 中等到高級
- **動機**: 管理社區資源，促進居民互動，與 NGO 合作建立交收點，將「書的流通」與「人的流通」結合
- **痛點**: 缺乏有效的社區資源管理工具

## 📖 **核心 User Stories**

### **Epic 1: 用戶帳戶管理 (User Account Management)**

#### **Story 1.1: 用戶註冊 (Sign Up)**
**作為** 新用戶  
**我想要** 創建一個新的 DLLM 帳戶  
**以便** 開始使用圖書共享服務

**用戶操作流程**:
1. 在首頁點擊「Sign Up」按鈕
2. 系統顯示註冊表單（模態對話框）
3. 填寫註冊信息：
   - **Email***: 輸入電子郵件地址（必填）
   - **Password***: 輸入密碼（必填，顯示為黑點）
4. 點擊「SIGN UP」按鈕提交註冊

**驗收標準**:
- [ ] 必填字段驗證和錯誤提示
- [ ] 響應式設計支持多種設備

**技術需求**:
- 電子郵件格式驗證

#### **Story 1.2: 用戶登入 (Sign In)**
**作為** 已註冊用戶  
**我想要** 登入我的 DLLM 帳戶  
**以便** 訪問個人圖書和借閱功能

**用戶操作流程**:
1. 在首頁點擊「Sign In」按鈕
2. 系統顯示登入表單（模態對話框）
3. 填寫登入信息：
   - **Email***: 輸入已註冊的電子郵件地址（必填）
   - **Password***: 輸入密碼（必填，顯示為黑點）
4. 點擊「SUBMIT」按鈕提交登入
5. 系統驗證用戶憑證
6. 登入成功後跳轉到用戶主頁
7. 如果忘記密碼，可點擊「FORGOT PASSWORD?」連結

**驗收標準**:
- [ ] 支持已註冊郵箱登入
- [ ] 支持密碼驗證
- [ ] 登入失敗時顯示友好錯誤信息
- [ ] 登入成功後正確跳轉
- [ ] 支持忘記密碼功能
- [ ] 響應式設計支持多種設備

**技術需求**:
- 用戶認證系統
- 密碼驗證和加密

#### **Story 1.3: 個人資料編輯 (Edit Profile)**
**作為** 已登入用戶  
**我想要** 編輯我的個人資料和設置位置信息  
**以便** 讓其他用戶了解我的基本信息和交收偏好

**用戶操作流程**:
1. 在用戶主頁點擊「Edit Profile」按鈕
2. 系統顯示個人資料編輯頁面
3. 編輯基本信息：
   - **Nickname***: 輸入或修改暱稱（必填）
   - **Address***: 輸入或修改地址（必填）
4. 系統顯示當前地址信息：「Current Address: [地址]」
5. 在地圖上顯示位置：
   - 地圖自動定位到輸入的地址
   - 顯示地圖標記指示位置
6. 確認位置設置後，點擊「UPDATE PROFILE」保存
7. 或點擊「CANCEL」取消編輯

**驗收標準**:
- [ ] 支持暱稱編輯和驗證
- [ ] 支持地址輸入和驗證
- [ ] 地圖自動定位到輸入地址
- [ ] 地圖標記準確顯示位置
- [ ] 支持地圖縮放和導航
- [ ] 保存和取消功能正常
- [ ] 響應式設計支持多種設備

**技術需求**:
- 地理編碼服務（地址轉座標）
- 地圖服務整合（OpenStreetMap）
- 個人資料數據管理
- 地址驗證和格式化
- 地圖組件和交互

### **Epic 2: 書籍上傳和放置 (Book Upload & Placement)**

#### **Story 2.1: 書籍註冊和基本信息錄入**
**作為** 書籍擁有者  
**我想要** 輕鬆註冊我的書籍到系統中  
**以便** 讓社區成員知道我有什麼書可以分享

**用戶操作流程**:
1. 登入 DLLM 系統
2. 點擊「Create Item」按鈕
3. 進入「Create Item」表單頁面，填寫以下必填和選填字段：
   **必填欄位**: 書名、分類（逗號分隔）、狀態（NEW, GOOD, FAIR, POOR）、語言（EN, ZH, JA 等）、可用狀態（AVAILABLE, UNAVAILABLE）
   **選填欄位**: 描述、上傳圖片、出版年份（如 2024）
4. 點擊「CREATE ITEM」按鈕完成書籍註冊，或點擊「CANCEL」取消操作

**驗收標準**:
- [ ] 支持書籍名稱輸入
- [ ] 支持分類標籤輸入（逗號分隔）
- [ ] 支持書籍狀態選擇（NEW, GOOD, FAIR, POOR）
- [ ] 支持語言選擇
- [ ] 支持可用狀態設置
- [ ] 支持可選描述輸入
- [ ] 支持圖片上傳功能
- [ ] 支持出版年份輸入（4位數格式）
- [ ] 表單驗證和錯誤提示
- [ ] 響應式設計支持多種設備

**技術需求**:
- 表單驗證和數據存儲
- 圖像上傳和處理
- 下拉選單組件
- 響應式表單設計
- 數據庫存儲和檢索

#### **Story 2.2: 書籍狀態管理和更新**
**作為** 書籍擁有者  
**我想要** 隨時更新書籍狀態和信息  
**以便** 保持書籍信息的準確性

**狀態與位置關聯**:
- **可用狀態**: 位置跟隨擁有者
- **借出狀態**: 位置自動更新為持有人位置
- **歸還狀態**: 位置自動恢復為擁有者位置

**用戶操作流程**:
1. 書籍擁有者(注: 不包括書籍持有人))可選擇自己擁有或持有的書籍
2. 點擊Edit Item
3. 可以更新的信息：
   - 書名
   - 書籍狀態（可用、借出、維護中、遺失等）
   - 個人描述和標籤
4. 系統自動記錄變更歷史
5. 如果書籍狀態變更為「借出」，系統自動：
   - 通知借書人
   - 更新書籍位置為持有人位置
   - 記錄位置變更歷史
6. 點擊 Update 完成

**驗收標準**:
- [ ] 實時狀態更新
- [ ] 位置自動跟隨狀態變化
- [ ] 變更歷史記錄
- [ ] 自動通知機制
- [ ] 批量操作支持

**技術需求**:
- 狀態管理系統
- 變更追蹤機制
- 實時通知推送
- 位置自動更新機制
- 狀態與位置關聯邏輯

### **Epic 3: 書籍借閱 (Book Borrowing)**

#### **Story 3.1: 書籍搜索和發現**
**作為** 借書人  
**我想要** 快速找到我需要的書籍  
**以便** 開始借閱流程

**用戶操作流程**:
1. 在首頁使用搜索功能：
   - 輸入關鍵詞（書名、作者、主題等）
   - 使用高級搜索選項（分類、語言、出版年份等）
   - 按地理位置篩選（距離、交通方式等）
2. 瀏覽搜索結果：
   - 查看書籍基本信息（標題、作者、封面、評分）
   - 查看可用性和位置信息
   - 查看借閱條件和期限
   - 查看書籍擁有者的評價
3. 使用篩選和排序功能：
   - 按距離排序
   - 按評分排序
   - 按可用性篩選
   - 按借閱條件篩選
4. 點擊感興趣的書籍查看詳細信息

**驗收標準**:
- [ ] 多種搜索方式支持
- [ ] 智能搜索建議
- [ ] 靈活的篩選和排序
- [ ] 搜索結果分頁顯示
- [ ] 搜索歷史記錄

**技術需求**:
- 全文搜索引擎
- 地理位置索引
- 智能推薦算法

#### **Story 3.2: 借閱申請和確認**
**作為** 借書人  
**我想要** 申請借閱書籍並獲得確認  
**以便** 開始使用書籍

**借閱流程概述**:
- **Borrower**: 發起 Request → 確認資料 → 按 Confirm Request
- **Owner**: 決定 Approve 或 Reject
- **Holder**: 決定 Approve 或 Reject（可能與 Owner 相同）
- **三方協調**: 通過 Email 溝通，私下決定交收時間和地點
- **交收完成**: Owner 按 Complete Transfer，Borrower 按 Received
- **狀態更新**: Current Holder 改為該 Borrower

**用戶操作流程**:

**Borrower 操作流程**:
1. 在書籍詳細頁面點擊「REQUEST」按鈕
2. 系統顯示「Confirm Request」對話框，包含：
   - 書籍基本信息（名稱、狀態、語言等）
   - 現有申請狀態檢查
   - 相關人員信息（Owner、Current Holder）
   - 交易地點信息
   - 通知說明
3. 確認資料正確後，點擊「Confirm Request」按鈕
4. 系統自動發送 Email 通知給 Owner 和 Current Holder
5. 等待 Owner 和 Holder 的審核決定

**Owner 操作流程**:
1. 收到借閱申請的 Email 通知
2. 在系統中查看「Pending Request」頁面
3. 查看申請詳情：
   - 申請人信息（姓名、聯繫郵箱）
   - 申請時間
   - 申請的書籍信息
4. 做出決定：
   - 點擊「APPROVE」按鈕批准申請
   - 點擊「REJECT」按鈕拒絕申請
5. 系統自動發送 Email 通知給其他相關者

**Holder 操作流程**:
1. 收到借閱申請的 Email 通知
2. 在系統中查看申請詳情
3. 做出決定：
   - 點擊「APPROVE」按鈕批准申請
   - 點擊「REJECT」按鈕拒絕申請
4. 系統自動發送 Email 通知給其他相關者

**三方協調階段**:
1. 當 Owner 和 Holder 都 Approve 後
2. 三方通過 Email 進行溝通
3. 私下決定：
   - 交收時間
   - 交收地點（家中、社區交收站等）
   - 交收方式

**交收完成階段**:
1. **Owner 操作**: 點擊「Complete Transfer」按鈕
2. **Borrower 操作**: 點擊「Received」按鈕
3. 系統自動更新：
   - 書籍狀態為「已借出」
   - Current Holder 改為該 Borrower
   - 書籍位置更新為 Borrower 位置
   - 記錄借閱開始時間

**驗收標準**:
- [ ] Borrower 可發起 Request 並確認資料
- [ ] 系統自動發送 Email 通知給 Owner 和 Current Holder
- [ ] Owner 可 Approve 或 Reject 申請
- [ ] Holder 可 Approve 或 Reject 申請
- [ ] 三方協調機制支持私下決定交收細節
- [ ] 交收完成後系統自動更新狀態和位置
- [ ] 完整的 Email 通知流程
- [ ] 申請狀態實時追蹤

**技術需求**:
- 申請流程管理系統
- Email 通知服務
- 狀態追蹤機制
- 三方協調支持
- 自動狀態更新機制
- 位置自動跟隨邏輯

#### **Story 3.3: 書籍交收和借閱開始**
**作為** 借書人  
**我想要** 順利完成書籍交收並開始借閱  
**以便** 使用書籍進行學習

**交收流程概述**:
- **三方協調**: Owner、Holder、Borrower 通過 Email 私下決定交收細節
- **交收完成**: Owner 按 Complete Transfer，Borrower 按 Received
- **狀態更新**: 系統自動更新書籍狀態、Current Holder 和位置

**用戶操作流程**:

**三方協調階段**:
1. 當 Owner 和 Holder 都 Approve 借閱申請後
2. 三方通過 Email 進行溝通協調
3. 私下決定交收細節：
   - 交收時間（具體日期和時間）
   - 交收地點（家中、社區交收站、約定地點等）
   - 交收方式（面對面、通過交收點等）
   - 聯繫方式和注意事項

**交收執行階段**:
1. 按約定時間和地點進行交收：
   - 如果是上門取書：按地址前往
   - 如果是約定地點：按約定前往
   - 如果是交收點：按開放時間前往
2. 完成實體交收後，雙方在系統中確認

**系統確認階段**:
1. **Owner 操作**: 點擊「Complete Transfer」按鈕
2. **Borrower 操作**: 點擊「Received」按鈕
3. 系統自動更新：
   - 書籍狀態為「已借出」
   - Current Holder 改為該 Borrower
   - 書籍位置更新為 Borrower 位置
   - 記錄借閱開始時間
   - 開始計算借閱期限

**借閱完成後**:
1. Borrower 成為新的 Current Holder，負責：
   - 回應其他用戶的借閱請求
   - 在需要時將書籍交給下一個借閱者
   - 按時歸還給原擁有者（如需要）

**驗收標準**:
- [ ] 支持三方 Email 協調交收細節
- [ ] 靈活的交收地點選擇（家中、交收點、約定地點）
- [ ] 雙向確認機制（Complete Transfer + Received）
- [ ] 系統自動更新書籍狀態和位置
- [ ] 自動計算借閱期限
- [ ] 智能提醒系統
- [ ] 支持「書主—持有人—借書人」的多點傳遞模式

**技術需求**:
- 三方協調支持系統
- Email 通知服務
- 交收流程管理
- 自動狀態更新機制
- 位置自動跟隨邏輯
- 時間計算引擎
- 提醒系統
