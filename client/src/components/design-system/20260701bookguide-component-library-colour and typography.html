Foundations — 01
Governing Colour Principle
One rule across all badge systems: colour tells users whether they can act. Build one mental model — users learn it once and it works everywhere in the app.
Green
Go — available or positively complete
Blue
Act — coordination or action required now
Amber
Wait — in motion, pending, or approaching limit
Red
Stop — blocked, overdue, expired or failed
Teal · Purple
Special — exchange offer, gift, or conditional access
Grey
Neutral — inactive, terminal, or holding
Always pair a badge with a role-specific action prompt. The badge tells users the state; the prompt tells them who acts next. Three amber transaction states look identical — only the prompt disambiguates them.

Bilingual Rules — 正體中文 + English
Rule 正體中文 (CJK) English (Latin)
Typeface Noto Serif TC IBM Plex Sans / Mono
Role Cultural voice — titles, greetings, nav, book names Functional voice — UI, body, metadata, badges
Min readable size 12px (stroke-dense glyphs) 9px (decorative only)
Body line-height 1.4–1.6 (extra leading for density) 1.5–1.6
Heading line-height 1.3 (tight) 1.2
Bold weight 700 600 (to match CJK 700 visually)
Mixed script rule: When CJK and Latin share a line, the CJK typeface governs. "5月35日" renders in Noto Serif TC, not Plex. Exception: system text — the tagline "Hong Kong books. Keep them moving." is Latin-primary, so Plex Mono governs.
Weight parity: CJK bold (700) reads heavier than Latin bold at the same weight. In mixed headings, use 600 for the Latin portion to match CJK 700 visually. Example: BookGuide (700) + Sydney (600).
CJK floor: Characters with 15+ strokes (鎮, 館, 灣, 歷) lose legibility below 12px on mobile. The 10px micro step is reserved for 1–2 character CJK chips (六四, 禁書) where glyphs are simple. Never set stroke-dense characters below 12px.
Weight Map
Weight Role

Accessibility — Contrast & Readability
All 23 text/background combinations pass WCAG AA. The majority hit AAA (7:1+). One constraint to enforce: ink-600 (#956a70) is the lightest text-safe colour — safe on white (4.6:1), borderline on ink-100 (4.2:1, passes only as large text ≥16px bold), and fails on ink-200 or darker. Never use ink-600 for must-read text below 16px bold on any surface darker than white.

4 · CJK / Latin Script Rules
These rules constrain the token system. An engineer picks a semantic token first, then checks these rules to confirm it's safe for the content script.
Rule 正體中文 (CJK) English (Latin)
Typeface --font-serif (Noto Serif TC) --font-sans / --font-mono
Voice Cultural — titles, greetings, nav, book names Functional — UI, body, metadata, badges
Min size (must-read) --fs-12 (12px) --fs-11 (11px)
Min size (decorative) --fs-10 (10px, 1–2 char only) --fs-9 (9px)
Body line-height --lh-normal (1.5) to --lh-relaxed (1.55) --lh-relaxed (1.55)
Heading line-height --lh-snug (1.3) --lh-tight (1.2)
Bold weight --fw-bold (700) --fw-semibold (600) in mixed headings
Mixed-script resolution: When CJK and Latin share a line, the CJK typeface (--font-serif) governs. Digits and punctuation within CJK text inherit the serif face. Exception: Latin-primary system text (tagline, version labels) uses --font-mono regardless of surrounding CJK context.
Weight parity: CJK glyphs at --fw-bold (700) appear heavier than Latin at the same weight due to stroke density. In mixed headings, pair CJK 700 with Latin 600 for visual balance. Example: BookGuide Sydney — the title is 700, the city is 600, but they read as the same visual weight.
CJK floor enforcement: Characters with 15+ strokes (鎮 館 灣 歷 讀 職 議 鐵) collapse below --fs-12 on mobile screens. The --type-micro step (10px) is safe only for simple CJK (六 四 書 人 文) — 1–2 character labels where every stroke is distinct. When in doubt, use --type-label (12px).
Accessibility gate: ink-600 (#956a70) is the lightest text-safe colour. It passes WCAG AA on white (4.6:1) and on ink-100 only at ≥16px bold. It fails on ink-200 or darker at any size. Never pair ink-600 text with --type-caption (11px) or --type-micro (10px) on any non-white surface.
